## 1. 项目概述

赛博公墓是一个 Web 应用，允许用户浏览、创建和编辑虚拟人物的纪念页面（墓志铭）。任何人都可以贡献内容，类似一个协作式的数字纪念碑。

### 核心功能

- **首页列表**：展示所有已收录人物，支持按分类筛选和关键词搜索。
- **人物详情**：查看完整信息（生平、墓志铭等）。
- **编辑人物**：修改已有信息。
- **新增人物**：创建新人物（可为自己撰写墓志铭）。

### 扩展目标（后续可选）

- 记录编辑历史（每次修改的昵称和时间）
- 简单防恶意编辑（如验证码或编辑锁）
- 图片上传功能

## 3. 数据库设计

使用 SQLite，只需一张表 `persons`。

### 表结构

```sql
CREATE TABLE IF NOT EXISTS persons (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  category TEXT DEFAULT 'other',
  avatar_url TEXT,
  bio TEXT,
  epitaph TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 字段说明

- `id`: 自增主键
- `name`: 人物姓名，必填
- `category`: 分类，可选值：`anime`（动漫）、`internet`（网红）、`fiction`（小说）、`real`（现实）、`self`（自建墓志铭）、`other`（其他）
- `avatar_url`: 头像图片 URL（可选）
- `bio`: 生平介绍（文本）
- `epitaph`: 墓志铭（文本）
- `created_at`: 创建时间
- `updated_at`: 最后更新时间（每次更新自动修改）

## 4. 后端 API 规范

### 基础 URL

所有 API 路径以 `/api` 开头。

### 接口列表

#### 4.1 获取所有人物

- **方法**: `GET`
- **路径**: `/api/persons`
- **查询参数**:
  - `search` (可选): 模糊匹配姓名
  - `category` (可选): 按分类筛选
- **响应**: JSON 数组，按 `updated_at` 降序排列
- **示例**: `/api/persons?search=李白&category=real`

#### 4.2 获取单个人物

- **方法**: `GET`
- **路径**: `/api/persons/:id`
- **响应**: 单个人物对象

#### 4.3 创建新人物

- **方法**: `POST`
- **路径**: `/api/persons`
- **请求体**: JSON，包含 `name`(必填)、`category`、`avatar_url`、`bio`、`epitaph`
- **响应**: 新创建的人物对象（含 `id`）

#### 4.4 更新人物

- **方法**: `PUT`
- **路径**: `/api/persons/:id`
- **请求体**: JSON，可更新任意字段
- **响应**: `{ updated: 1 }` 表示成功

#### 4.5 删除人物（可选）

- **方法**: `DELETE`
- **路径**: `/api/persons/:id`
- **响应**: `{ deleted: 1 }`

## 5. 前端页面设计

### 页面结构

项目根目录下 `public` 文件夹，包含以下页面：

- `index.html` – 首页列表
- `detail.html` – 详情页
- `edit.html` – 编辑/新增页
- `style.css` – 全局样式
- `script.js` – 公共函数（可选）

### 页面功能细节

#### 5.1 首页 (`index.html`)

- 顶部导航栏：标题“赛博公墓”、搜索框、分类下拉菜单、一个“给自己写墓志铭”按钮。
- 人物卡片列表：每个卡片包含头像（若无则占位）、姓名、分类徽章、墓志铭预览（截断）。
- 点击卡片跳转到 `detail.html?id=xxx`。
- 搜索和筛选触发重新加载列表（向后端请求）。

#### 5.2 详情页 (`detail.html`)

- 根据 URL 参数 `id` 获取人物数据。
- 展示完整信息：姓名、分类、头像（大图）、生平、墓志铭（突出样式）。
- 底部显示“最后更新于”时间。
- 提供“编辑”按钮，跳转到 `edit.html?id=xxx`。
- 若数据加载失败，显示错误提示。

#### 5.3 编辑/新增页 (`edit.html`)

- 根据 URL 参数 `id` 判断模式：
  - 有 `id`：编辑模式，先加载现有数据并填充表单。
  - 无 `id`：新增模式，表单为空。
- 表单包含：姓名（文本框，必填）、分类（下拉菜单）、头像 URL（文本框）、生平（文本域）、墓志铭（文本域）。
- 提交按钮：保存后根据模式调用相应 API，成功后跳转回详情页（编辑模式）或首页（新增模式）。
- 取消按钮：返回上一页。

## 8. 安全与注意事项

- **SQL 注入防护**：始终使用参数化查询（`db.run`、`db.all` 的 `?` 占位符）。
- **XSS 防护**：前端渲染用户输入内容时，使用 `textContent` 或进行 HTML 转义。
- **输入验证**：后端应验证 `name` 不为空，`category` 是否为允许的值等。
- **错误处理**：所有 API 返回适当的 HTTP 状态码（200、400、404、500）及错误信息。

  if (err) console.error('数据库连接失败', err);
  else console.log('数据库连接成功');

db.serialize(() => {

      epitaph TEXT,

### 9.3 前端提交示例 (`edit.html`)

```javascript
document.getElementById('edit-form').addEventListener('submit', async (e) => {
  e.preventDefault();
  const formData = new FormData(e.target);
  const data = Object.fromEntries(formData.entries());
  const id = new URLSearchParams(window.location.search).get('id');
  const url = id ? `/api/persons/${id}` : '/api/persons';
  const method = id ? 'PUT' : 'POST';
  const res = await fetch(url, {
    method,
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(data)
  });
  if (res.ok) {
    alert('保存成功');
    window.location.href = id ? `detail.html?id=${id}` : 'index.html';
  } else {
    alert('保存失败');
  }
});
```
